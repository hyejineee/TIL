# Mental Model

> "A compressed model based on what we think we know about a system and how it works."

## 한 줄 정리

사용자는 머릿속에 작은 시뮬레이션을 가지고, 새 시스템에 그걸 그대로 적용한다. 그 모델과 실제 동작이 어긋나면 혼란이 생긴다.

## 핵심 원리

1. 사용자는 시스템이 어떻게 동작하는지 머릿속에 작동 모델을 만들고, 비슷한 새 상황에 그걸 적용한다.
2. 제품 디자인을 사용자의 기존 멘탈 모델에 맞추면 — 학습 비용 없이 — 다른 제품에서 쌓은 지식이 그대로 전이된다.
3. 사용자 기대와 디자인이 일치하면 강력한 UX. e-커머스의 product card·장바구니·체크아웃 흐름이 표준화된 이유.
4. **디자이너 멘탈 모델 ↔ 사용자 멘탈 모델의 격차를 좁히는 것**이 UX의 핵심 과제 — 인터뷰·페르소나·저니맵·empathy map으로 접근.

## 출처

Kenneth Craik, *The Nature of Explanation* (1943). "사람은 머릿속에 세상이 어떻게 동작하는지의 작은 모델을 들고 다닌다"는 이론.

## 적용

- **컨벤션 따르기**: 일반적인 패턴(우상단 장바구니, 좌측 메뉴)을 깨려면 이유가 있어야 한다 → Jakob's Law와 연결.
- **에러 메시지**: "왜 이런 결과가 나왔는지" 사용자 멘탈 모델과 시스템 동작의 불일치를 메우는 단서.
- **사용자 인터뷰**: 사용자가 시스템을 어떻게 "설명"하는지 들어보면 멘탈 모델이 드러난다.
- **새 기능 도입**: 기존 멘탈 모델을 활용하는 메타포(예: "휴지통", "폴더", "북마크")가 학습 비용을 낮춘다.
- **온보딩**: 사용자의 가설을 검증·교정할 수 있는 작은 인터랙션을 초반에 배치.
