# Flow

> "The mental state in which a person performing some activity is fully immersed in a feeling of energized focus, full involvement, and enjoyment in the process of the activity."

## 한 줄 정리

활동에 완전히 몰입한 정신 상태. 작업 난이도와 사용자 능력의 균형에서 나온다.

## 핵심 원리

1. **균형**: 난이도가 능력과 맞을 때 몰입이 일어난다. 강한 집중과 활동에 대한 통제감이 동반된다.
2. **불균형의 경고**: 너무 어려우면 좌절, 너무 쉬우면 지루함. 둘 다 부정적 경험.
3. **피드백 메커니즘**: 사용자가 액션을 완료했고 목표에 가까워지고 있다는 신호를 제공.
4. **반응성·마찰 제거**: 시스템 응답이 빠르고 마찰이 적을수록 몰입이 유지된다.

## 출처

Mihály Csíkszentmihályi (1975). 미치이라고 발음. 다양한 분야 — 특히 작업 치료(occupational therapy) — 에서 인정받음.

## 적용

- **온보딩**: 사용자 능력에 맞춰 점진적으로 난이도를 올린다 (Slack의 progressive onboarding).
- **즉각 피드백**: 입력 → 변화가 보여야 한다. 토글이 부드럽게 전환되고, 검색 결과가 즉시 갱신되고.
- **마찰 제거**: 불필요한 confirm 모달, 광고, 인터럽트를 제거. 매끄러운 흐름.
- **단계적 도전**: 게임에서 흔한 "easy → medium → hard"가 일반 SaaS 튜토리얼에서도 통한다.
- **상태 보존**: 작업 중 컨텍스트가 끊기면 몰입이 깨진다 — 자동 저장, 복원, undo.
