## ⚠️ 1차 집계가 틀렸던 이유 — Git author 변종 함정

처음 `--author="Hyejin"`으로 매칭했을 때 **modoogram 50건**이 나와 "이상한데?"라는 체감과 어긋났음. 원인은 본인 commit이 **5가지 author 변종**으로 나뉘어 있던 것:

| Author 표기 | Email | 어디서 push |
|---|---|---|
| `Hyejin Hyun` | dev.hyejin04@gmail.com | 메인 머신 (영문) |
| `Hyejin Hyun` | 44992033+hyejineee@users.noreply.github.com | GitHub 웹 squash-merge |
| `현혜진` | dev.hyejin04@gmail.com | 동일 머신 (한글 이름 설정) |
| `현혜진` | hyeonhyejin@hyeonhyejin-ui-MacBookAir.local | 다른 MacBook |
| `hyejin` | hyejin@hyejinui-MacBookPro.local | 또 다른 MacBook |

→ `--author="Hyejin Hyun"` 단일 패턴은 50건만 매칭. 한글·소문자 author와 squash-merge 시 GitHub noreply 이메일로 박힌 commit이 통째로 빠짐. 정정된 수치는 본인 5가지 변종 OR 합산 + `gh pr list --author=@me` 로 확보. (브랜치 삭제 의심은 빗나감 — `git log --all` 자체는 reflog/원격 ref까지 다 잡았음.)
